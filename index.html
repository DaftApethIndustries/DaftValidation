The requiredIf rule checks if another item has a value and if it DOES requires itself to be filled in.

The requiredIfNot rule checks if another item has a value and if it DOES NOT requires itself to be filled in.

The password rule validates the data is over 8 characters, contains upper and lower case and either numerals or symbols.

Test that the value matches a valid uk postcode
Accepted formats: 1. LN NLL eg N1 1AA 2. LLN NLL eg SW4 0QL 3. LNN NLL eg M23 4PJ
4. LLNN NLL eg WS14 0JT 5. LLNL NLL eg SW1N 4TB 6. LNL NLL eg W1C 8LQ

Add a new rule

		var exampleNewValidation = function (eField, optionalParam) {
			//perform validation
			//return false if it fails
			return true; //if it passes
		};

		var newSuccessCallback = function (eField) { };

		/**
		* Set a validation rule. rootObject name will be your namespace
		* Name will be how it is referred to in the data-validation rules array (here NewValidation)
		* Function is the function used to validate the data (here exampleNewValidation)
		*/
		looseValidation.addValidationFunction('NewValidation', exampleNewValidation);

		/**
		* Set a success function. rootObject name will be your namespace
		* Name will be how it is referred to in the data-success array (here NewSuccess)
		* Function is the function to call (here newSuccessCallback)
		*/
		looseValidation.addSuccessFunction('NewSuccess', newSuccessCallback);


		/*** USAGE ***/
		data-validation='{"rules":["NewValidation"],"messages":["Please fill this in"]}'
		data-success='{"func":"NewSuccess"}'
